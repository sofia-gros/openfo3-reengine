# ノーマルマップ / スペキュラーマップ対応 (#12)

## 問題
NIFMaterialBuilder が FO3 のマテリアルテクスチャを不正確にマッピングしていた。

### 具体的な問題

1. **Roughness が反転** — ノーマルマップのアルファチャンネルに格納された Gloss（高い＝光沢）をそのまま Roughness として使用していた。Godot の Roughness は低い＝光沢であり、結果が反転していた。
2. **SpecularMode が常に有効** — FO3 の ShaderFlags ビット 0 (Specular) の状態にかかわらず常に SchlickGGX だった。Specular フラグが立っていないマテリアルでも不要なスペキュラハイライトが表示されていた。
3. **Roughness のソースが不適切** — ノーマルマップアルファから直接読み取っていたが、これは FO3 の Gloss（0=rough, 1=glossy）であり、Godot の Roughness（0=glossy, 1=rough）と逆。

## 修正内容

### roughness マッピングの修正
- ノーマルマップのアルファチャンネルからの Roughness 読み取りを削除
- 代わりに Environment Mask (Slot 5) の赤チャンネルを Roughness テクスチャとして使用
- デフォルトの Roughness = 0.6, Metallic = 0.0 を設定

### SpecularMode の制御
- ShaderFlags ビット 0 (Specular) が立っている場合のみ `SpecularMode.SchlickGgx` を有効化
- 立っていない場合は `SpecularMode.Disabled` に設定

### Environment Mask (Slot 5) の活用
- FO3 の環境マップマスク（Slot 5）はスペキュラ強度・グロス情報を含むため、Roughness テクスチャとして利用
- 赤チャンネルを RoughnessTextureChannel に指定

## FO3 テクスチャスロットと Redot マッピング

| Slot | FO3 用途                | Redot マッピング                |
|------|--------------------------|----------------------------------|
| 0    | Diffuse (拡散)           | AlbedoTexture                   |
| 1    | Normal + Gloss (法線+グロス) | NormalTexture (gloss は破棄)  |
| 2    | Glow (発光)              | EmissionTexture                 |
| 3    | Height / Parallax        | HeightmapTexture                |
| 4    | Environment (環境)       | MetallicTexture (Red)           |
| 5    | Environment Mask (環境マスク) | RoughnessTexture (Red)      |
| 6    | Subsurface / Detail      | DetailAlbedo                    |

## 残課題
- FO3 の Gloss→Roughness の正確な変換（反転）にはカスタムシェーダーが必要。現時点では Environment Mask で代用
- FO3 の Specular Power (NiSpecularProperty) は PBR に直接マッピングできない。必要に応じてカスタムシェーダーで対応
- Parallax / Detail テクスチャはロードされているが、表示のチューニングは未調整
