# NIFMaterialBuilder.cs 構造破損修正

## 問題

`NIFMaterialBuilder.cs` に深刻な構造破損があり、ビルドが 23 エラーで失敗していた。

### 具体的な問題点

1. **BuildDefault メソッドのブレース不一致** (line 159):
   ```csharp
   if ((shader.ShaderFlags & 0x00000001) != 0)
       mat.SpecularMode = ...;
   }  // ← 単一行if文なのに閉じブレースがあり、else が追い出されていた
   else
   ```
   - 単一行 if 文に `}` が付いており、メソッドがそこで閉じてしまい、`else` がクラス直下の浮遊コードになっていた

2. **BuildEnvironmentMap 後の浮遊コード** (lines 248-281):
   - BuildEnvironmentMap の `}` 以降に、本来メソッド内にあるべき Heightmap、Detail、Refraction のコードブロックが浮遊
   - これらのコードは `texPaths` と `shader` 変数を参照していたが、どのメソッドのスコープにも属していなかった

3. **ApplyRefraction メソッドの欠落**:
   - `BuildMaterial` 内で `ApplyRefraction(mat, shader)` を呼び出していたが、対応するメソッドが存在しなかった

4. **ApplyCommonTextures のパラメータ不足**:
   - `shader.ShaderFlags` を使用していたが、`shader` パラメータがメソッドシグネチャに含まれていなかった

## 修正内容

### ファイル全体の再構成

```
BuildMaterial()                          ← エントリポイント
├── ApplyAlpha()                         ← アルファテスト/ブレンド
├── BuildGlowShader()                    ← type 3
├── BuildEnvironmentMap()                ← type 2, 17
├── BuildSkinTint()                      ← type 31
├── BuildHairTint()                      ← type 32
├── BuildTallGrass()                     ← type 34
├── BuildMultiLayerParallax()            ← type 10, 11, 15, 33
├── BuildHeightmap()                     ← type 4
├── BuildDefault()                       ← その他すべて
├── ApplyHeightmap()                     ← Slot 3 (共通処理)
├── ApplyDetail()                        ← Slot 6 (共通処理)
└── ApplyRefraction()                    ← ShaderFlags ビット15/16
```

### 具体的な修正

1. **BuildDefault**: `}` を削除し、`if-else` を正しい構文に修正
2. **浮遊コード** → `ApplyHeightmap()`, `ApplyDetail()`, `ApplyRefraction()` として独立した private static メソッドに分離
3. **ApplyRefraction** メソッドを新規作成（浮遊コードから移行）
4. **ApplyCommonTextures**: `ShaderTextureInfo shader` パラメータを追加
5. **BuildMaterial**: 各 BuildXxx メソッドの呼び出し後に `ApplyHeightmap`, `ApplyDetail`, `ApplyRefraction` を呼び出すよう修正
6. **SkinTint の API**: Redot 26.1 の API に合わせて `SubsurfScatterEnabled` / `SubsurfScatterStrength` / `SubsurfScatterSkinMode` / `SubsurfScatterTransmittanceEnabled` / `SubsurfScatterTransmittanceColor` を使用

### Redot 26.1 の Subsurface API 対応

| プロパティ (Godot 4) | Redot 26.1 のプロパティ |
|---|---|
| `SubsurfaceMode = SubsurfaceModeEnum.Skin` | `SubsurfScatterEnabled = true`<br>`SubsurfScatterSkinMode = true` |
| `SubsurfaceStrength` | `SubsurfScatterStrength` |
| `SubsurfaceTransmittanceEnabled` | `SubsurfScatterTransmittanceEnabled` |
| `SubsurfaceTransmittanceColor` | `SubsurfScatterTransmittanceColor` |

## 結果

- ビルド: 23 errors → **0 errors, 3 warnings** (既存)
- 構造破損なし、すべてのメソッドが適切に配置
- 全シェーダータイプのディスパッチが正常に機能

## 影響範囲

`NIFMaterialBuilder.cs` のみ。インターフェース（`BuildMaterial` のシグネチャ）は変更なし。
