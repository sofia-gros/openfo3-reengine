# 頂点カラーの抽出と非地形メッシュへの適用

## 問題

FO3 の NIF ジオメトリデータ（NiGeometryData）は `HasVertexColors` フラグと頂点カラーデータ（RGBA 4×float = 16 bytes/vertex）を持つ。従来のコードでは頂点カラーを読み飛ばし（`ms.Seek(numVertices * 16, ...)`）、メッシュ構築時に無視していた。

## 影響

- **SkinTint / HairTint シェーダー**: `VertexColorUseAsAlbedo = true` が有効でも、頂点カラーがメッシュに含まれていないためティント効果が発現しなかった
- **地形**: TerrainBuilder は VCLR サブレコードから独自に頂点カラーを適用しているため影響なし
- **一部の STAT メッシュ**: 頂点カラーによる色調補正が適用されず、見た目が不正確だった

## 修正内容

### 1. NiTriStripsDataParser / NiTriShapeDataParser

両パーサーの戻り値に `Color[] VertexColors` を追加。4連続 float（RGBA）を読み取る:

```csharp
bool hasVertexColors = br.ReadByte() != 0;
Color[] vertexColors = null;
if (hasVertexColors)
{
    vertexColors = new Color[numVertices];
    for (int i = 0; i < numVertices; i++)
    {
        float r = br.ReadSingle();
        float g = br.ReadSingle();
        float b = br.ReadSingle();
        float a = br.ReadSingle();
        vertexColors[i] = new Color(r, g, b, a);
    }
}
```

### 2. NIFGeometryData

Surface タプルに `Color[] VertexColors` フィールドを追加:

```csharp
public List<(Vector3[] Vertices, Vector2[] UVs, int[] Indices, Transform3D Transform,
    string TexturePath, ShaderTextureInfo Shader, AlphaPropertyInfo Alpha,
    Color[] VertexColors)> Surfaces = new();
```

### 3. NIFMeshBuilder.TraverseExtract

パーサーの戻り値から頂点カラーを取得し、`geom.Surfaces.Add` に渡す。

### 4. NIFMeshBuilder.BuildArrayMesh

SurfaceTool で各頂点に `SetColor()` を呼び出す:

```csharp
if (surface.VertexColors != null && vi < surface.VertexColors.Length)
    st.SetColor(surface.VertexColors[vi]);
```

### 5. NIFCollisionBuilder

頂点カラー追加によるタプル変更に対応。衝突形状ビルダーの `NiTriStripsDataParser.Parse` 呼び出しを 4 要素タプルに更新。

## FO3 頂点カラーの用途

FO3 の頂点カラーは主に以下の目的で使用される:

| 用途 | 説明 |
|------|------|
| SkinTint ティント | NPC の肌色/血色（赤みの頂点カラー） |
| HairTint ティント | 髪の色のバリエーション |
| 環境光のベイク | 事前計算されたライトマップ代わり |
| 反射/発光の強度制御 | Albedo に乗算して色合いを調整 |
