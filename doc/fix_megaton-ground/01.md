# メガトンの地面がない問題 (#10) 修正

## 作業内容

MegatonWorld の地面が表示されない問題を修正した。

## 原因

MegatonWorld は内装ワールドスペースであり、FO3 の LAND レコード（高マップ地形データ）を持たない可能性が高い。以下の複合的な問題があった:

1. **LAND レコード不在時のフォールバック欠落**:
   - `BuildTerrainForWorld` はまず LAND レコードを検索し、なければ gap fill（平坦地形補完）に頼る
   - この gap fill は `coveredCells`（実際に地形タイルが生成されたセル）の範囲をもとに補完範囲を決定する
   - `coveredCells` が空（LAND レコードがない、または CELL 座標が取得できない）の場合、`landMinX` が `int.MaxValue` のままとなり、補完範囲が `int.MaxValue..int.MinValue` になる → ループが実行されない
   - 結果として平坦地形が一切生成されず、地面が完全に欠落する

2. **CELL 座標 (XCLC) の不在**:
   - 内装ワールドの CELL レコードは XCLC サブレコード（外部セル座標）を持たない
   - `TryGetCellCoord` が失敗し、LAND レコードがあっても位置特定不可でスキップされる

3. **MNAM セル範囲が狭すぎる/未設定**:
   - WRLD レコードの MNAM データが未設定（0,0-0,0）の場合、地形生成範囲が 1 セル分しかなくなる

## 修正内容

### TerrainBuilder.cs

1. **強制平坦地形生成の追加** (force-generate):
   - すべての LAND タイル/gap fill が失敗した場合（`tiles.Count == 0`）、最終フォールバックとしてワールドのセル範囲全体に平坦地形を強制生成する
   - `halfSize = 70` の制限で最大 141x141 = 19,881 セルまで対応（実用的には十分）

2. **gap fill の改良**:
   - `fillCellMin/Max` が未設定（デフォルト 0,-1）でも、`coveredCells` が空でなければ自動的に範囲を計算
   - `hasValidBounds` 判定の改善

### Megaton.cs

1. **MNAM 範囲の自動拡張**:
   - MNAM が未設定（0,0-0,0）でも、最低 15x15 セルの地形範囲を確保
   - 中心からのハーフサイズ: `max((se-nw)/2, 10) + 5` セル

## 期待される動作

- MegatonWorld に LAND レコードがなくても、デフォルト陸面高さで平坦な地面が表示される
- 地面の色は薄い緑褐色 (RGB 0.45, 0.55, 0.35)
- 地面には ConcavePolygonShape3D のコリジョンが設定される

## 今後の課題

- 床用 STAT メッシュ（地面の見た目を向上させる静的メッシュ）の抽出は未対応
- 複数ワールド (Wasteland 等) への対応は別 Issue
