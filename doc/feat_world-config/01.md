# feat_world-config: ワールド設定の外部化

## 概要
Megaton.cs にハードコードされていた "MegatonWorld" の選択と
中心座標 (-14200, -3800) を config.json の World セクションで
設定可能にした。

## 変更内容

### config.json
```json
"World": {
  "TargetWorld": "MegatonWorld",
  "CenterX": -14200.0,
  "CenterY": -3800.0
}
```
- **TargetWorld**: WRLD レコードの EDID 名（例: "Wasteland", "DCRuins"）
- **CenterX/CenterY**: FO3 ワールド座標系の中心オフセット

### GamePaths.cs
- `_targetWorld` / `_worldCenterX` / `_worldCenterY` 静的フィールド追加
- Initialize() で "World" セクションを読み取り
- `GetTargetWorld()` / `GetWorldCenter()` 公開メソッド追加

### Megaton.cs
- `_Ready()` で `GamePaths.GetTargetWorld()` と `GamePaths.GetWorldCenter()` を使用
- WRLD 検索ループの比較文字列をハードコードから config 値に変更
- デバッグメッセージも動的名前に

## NIFBlockResolver 追記
ついでに NiBillboardNode / NiSortAdjustNode をノード種別判定に追加。
これまで未対応だったため、これらのノードを含む NIF（TREE の葉など）の
子階層が適切にトラバースされていなかった。
