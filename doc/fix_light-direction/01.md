# ライトの向きが正しくない問題 (#11) 修正

## 作業内容

FO3 の LIGH レコードから Redot Engine の Light3D ノードに変換する処理のバグを修正した。

## 原因

### 1. 位置の二重変換（クリティカルバグ）

`Megaton.cs::ProcessRecord()` は REFR の DATA サブレコードから位置を読み取り、Godot 座標に変換する:
```csharp
Position = new Vector3((px - _megatonCenter.X) * WorldScale,
                       pz * WorldScale,
                       -(py - _megatonCenter.Y) * WorldScale);
```

これはメッシュインスタンスで使用される `req.Position` と同一（既に Godot 座標）。

しかし `LightingLoader.CreateLightNode()` は受け取った `position` に対して**さらに別の変換**を適用していた:
```csharp
float px = position.X;
float py = position.Z;
float pz = -position.Y;
light.Position = new Vector3(px, py, pz);
```

結果: ライトの位置がメッシュの位置とズレていた。

### 2. 減衰率とエネルギーの未反映

すべてのライトで `LightEnergy = 1.0f` に固定。FO3 の `FalloffExponent`（減衰率）が無視されていた。

### 3. 範囲の下限が不適切

```csharp
omni.OmniRange = Mathf.Max(radius, 1f);
```
小さな光源でも強制的に range=1.0 に拡大されていた。

## 修正内容

1. **位置変換の修正**: 二重変換を削除し、`position` を直接 `Transform3D` に使用
2. **減衰率とエネルギーの計算**:
   - `FalloffExponent` → `Attenuation = clamp(falloff * 0.5, 0.5, 4.0)`
   - `FalloffExponent` → `LightEnergy = clamp(1.0 / falloff, 0.3, 2.0)`
3. **範囲下限の緩和**: 最小 0.5 Godot 単位

## 結果

- ライトの位置がメッシュと一致
- 減衰率が FO3 データに基づいて設定
- 小さな光源が適切な範囲で表示

## 残課題

- ライトにカメラが埋まったときの表示問題は Godot エンジンのレンダリング特性の可能性
- セルライティング（XCLL）の方向変換は検証不足
- FO3 シャドウマップシステムの再現は未実装
