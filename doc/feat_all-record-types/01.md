# 全レコードタイプ対応

## 背景

従来は `_masterFormIDIndex` に `STAT` と `DOOR` のみ登録されており、その他のレコードタイプ（FURN, ACTI, MSTT, LIGH 等）を参照する REFR はすべてスキップされていた。これによりメガトン内部で大量のオブジェクトが描画されていなかった。

## 変更内容

### 1. 全レコードタイプを index に追加

`Megaton.cs:75-78` の `_masterFormIDIndex` に以下のレコードタイプを追加:

```
STAT, DOOR, FURN, ACTI, MSTT, LIGH, TERM, CONT, MISC, WEAP,
ARMO, CLOT, TREE, ALCH, INGR, BOOK, DEBR, SCOL
```

除外したタイプ:
- `LAND` — 地形は TerrainBuilder が専用に処理
- `GRAS` — 草は頂点配置方式のため別処理が必要

### 2. InstanceRequest に BaseType を追加

`ProcessRecord` 内でベースレコードのタイプを取得し `InstanceRequest.BaseType` に保存。これにより `CreateAndAddInstance` で再取得するための余分な ESM 読み取りが不要になった。

### 3. LIGH の MODL 不在ケースに対応

光源のみでメッシュ（灯具）を持たない LIGH レコードがある。従来は `modl == null` で早期 return していたため、そうした REFR は完全に無視されていた。

修正後:
- `nifPath == null && baseType != "LIGH"` → return（通常オブジェクトは従来通り）
- `nifPath == null && baseType == "LIGH"` → NIF の読み込みをスキップし、InstanceRequest をキューに投入
- `CreateAndAddInstance` で `BaseType == "LIGH"` の場合にライトノードを作成

### 4. デッドコード削除

`IsLightRef()` / `GetBaseObjectType()` を削除。`BaseType` を直接使うようになったため不要。

## 残課題

### レコードタイプ別の特殊処理
現状は全タイプを同じ汎用パス（REFR → MODL → NIF → Mesh）で処理している。以下のタイプは将来的に特殊処理が必要:

| タイプ | 必要な処理 |
|--------|------------|
| LIGH | 灯具メッシュとは別に OmniLight3D/SpotLight3D を作成中（済み） |
| SCOL | 複数の STAT を参照するコレクション。NIF はコレクション全体のメッシュなので現状でも動作する |
| TREE | FO3 独自の SpeedTree 形式の可能性。MODL がない場合あり |
| GRAS | 草の頂点配置方式。別途実装が必要 |
| CONT, WEAP, MISC 等 | インタラクション（開閉/取得）は未実装。描画のみ対応 |

### パフォーマンス
- インデックスサイズ増大による ESM 読み取り時間増加
- 描画オブジェクト数増加によるフレームレート低下
- 必要に応じて `MaxObjectsToLoad` の調整や LOD 導入を検討
