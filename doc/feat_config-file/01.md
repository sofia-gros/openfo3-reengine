# 設定ファイル化 (GamePaths + config.json)

## 問題

`GamePaths.cs` の Fallout 3 インストールパスがハードコードされている。他の環境でビルドするたびにソースコードの編集が必要だった。

## 修正内容

### 1. config.json の新規作成

プロジェクトルートに `config.json` を追加:

```json
{
  "Fallout3Root": "A:\\SteamLibrary\\steamapps\\common\\Fallout 3 goty",
  "DataPath": "",
  "EsmPath": "",

  "BSA": {
    "Meshes": "Fallout - Meshes.bsa",
    "Textures": "Fallout - Textures.bsa",
    "Misc": "Fallout - Misc.bsa",
    "Voices": "Fallout - Voices.bsa"
  },

  "Rendering": {
    "WorldScale": 0.015,
    "ShadowAtlasSize": 4096,
    "DirectionalShadowSize": 4096,
    "SoftShadowQuality": 1
  }
}
```

- `Fallout3Root` が設定されている場合はそのパスを使用
- 未設定・ファイル不在の場合はデフォルトパスにフォールバック
- `DataPath` / `EsmPath` は `Fallout3Root` から自動構築（上書き可能）
- BSA/レンダリング設定は将来の拡張用（現在未使用）

### 2. GamePaths.cs の改修

- シングルスレッドセーフな遅延初期化パターンに変更
- `config.json` を `ProjectSettings.GlobalizePath("res://config.json")` で検索
- JSON パースに `System.Text.Json` を使用（.NET 8 標準機能、追加パッケージ不要）
- エラー時はデフォルトパスにフォールバック

```csharp
// 使用例:
string esmPath = GamePaths.EsmPath;
// => 設定ファイルから読み込まれたパス、
//     またはデフォルト値
```

### 3. 動作

- ビルド成果物に `config.json` を同梱する必要がある（Godot の res:// 経由のため自動的）
- 設定が存在しない場合、元のハードコードされたパスが使用される（後方互換性維持）

## 動作確認

- `dotnet build` 正常終了 (0 errors, 3 pre-existing warnings)
