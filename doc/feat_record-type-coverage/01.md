# レコードタイプ対応拡充

## 問題

`Megaton.cs` の `_masterFormIDIndex` が一部の REFR 参照先レコードタイプを網羅していなかった。REFR が未登録のタイプを参照している場合、`_masterFormIDIndex.TryGetValue` が失敗し、該当インスタンスが無視されていた。

## 修正内容

### インデックスに追加したレコードタイプ

| タイプ | 説明 | FO3 での MODL 有無 |
|--------|------|-------------------|
| KEYM | 鍵 (Key) | MODL あり（キーピック等） |
| ARMA | 防具追加パーツ (Armor Addon) | MODL あり |
| NOTE | 手紙/メモ (Note) | MODL あり |
| PWAT | 設置可能な水 (Placeable Water) | MODL あり |
| TACT | 会話型アクティベータ (Talking Activator) | MODL あり |
| AMMO | 弾薬 (Ammunition) | MODL あり |

これらのタイプはすべて FO3 の REFR レコードが NAME サブレコードで参照し得るベースオブジェクトであり、かつ MODL (Model) サブレコードを持つ。

### 現在のマスターインデックス完全リスト

```
STAT, DOOR, FURN, ACTI, MSTT, LIGH, TERM, CONT, MISC,
WEAP, ARMO, CLOT, TREE, ALCH, INGR, BOOK, GRAS, LAND,
DEBR, SCOL, KEYM, ARMA, NOTE, PWAT, TACT, AMMO
```

全 26 タイプ。

### 既存コードでの処理フロー

全タイプは `ProcessRecord()` で同一の処理パスを通る:

```
REFR の NAME → _masterFormIDIndex 検索 → MODL から NIF パス取得 →
非同期 NIF パース & メッシュ生成 → CreateAndAddInstance で配置
```

LIGH タイプのみ、メッシュ配置後に `CreateLightForRef()` で Light3D ノードを追加する分岐がある。

## 未対応の REFR 参照先タイプ

| タイプ | 理由 | 対応予定 |
|--------|------|---------|
| SOUN | サウンド（視覚オブジェクトではない） | 未定（音声システム実装時） |
| LVLN/LVLC | レベリング済み NPC/クリーチャー（NPC システム） | NPC 実装時 |
| ASPC | 音響空間（オーディオのみ） | 未定 |
| IDLM | アイドルマーカー（マーカーエディタ） | 低優先度 |
| TXST | テクスチャセット（オブジェクトではない） | 不要 |
| SCOL | 静的コレクション（複数メッシュの集合） | 別途対応予定 |
