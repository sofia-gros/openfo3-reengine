# feat_scol-parts: SCOL パーツ配置の実装

## 背景
SCOL (Static Collection) は複数の STAT インスタンスを1つの配置としてまとめたレコード。
従来は MODL フォールバック（SCOL 自身のメッシュのみ表示）で暫定対応していた。

## 変更内容

### Megaton.cs
- `InstanceRequest` に `Scale` フィールドを追加
- `ProcessRecord`: SCOL の場合 `EmitScolParts()` を呼び出し
- `EmitScolParts()`: ONAM + DATA サブレコードのペアを走査
  - ONAM: STAT の FormID → マスターインデックスから MODL パスを解決
  - DATA: 可変長の配置配列（7 float: X, Y, Z, RX, RY, RZ, Scale）
  - SCOL のワールド変換行列と Part のローカル変換を合成
  - 合成後の位置/回転で個別 InstanceRequest を生成
- `CreateAndAddInstance`: Basis.Scaled() でスケールを適用

### 座標変換
SCOL Part のローカル座標をワールド座標に変換:
1. FO3 回転行列 R = Rz(Z軸回転) * Ry(Y軸回転) * Rx(X軸回転) を構築
2. `world_pos = scol_pos + R * local_pos`
3. `world_rot = scol_rot + local_rot`（簡易合成、精度は実用レベル）

### 確認
- 0 errors, 0 warnings
- SCOL の各 Part が正しく STAT メッシュとして配置される
