# シャドウマップの再現

## 問題

すべての光源で `ShadowEnabled = false`（Godot デフォルト）だった。影が一切描画されず、立体感・奥行き感が乏しかった。

## 修正内容

### 1. DirectionalLight3D（太陽）にシャドウカスケードを設定

`LightingLoader.ApplyCellLighting()` (`LightingLoader.cs:219-229`):

| プロパティ | 値 | 説明 |
|---|---|---|
| `ShadowEnabled` | `true` | 影を有効化 |
| `ShadowBias` | `0.001` | ワールドスケール 0.015 に合わせて調整 |
| `ShadowNormalBias` | `0.01` | 法線バイアスも同様に調整 |
| `DirectionalShadowMaxDistance` | `200` | 影が届く最大距離 (Godot 単位) |
| `DirectionalShadowBlendSplits` | `true` | カスケード分割境界のブレンド有効化 |

PSSM (Parallel Split Shadow Map) は Godot 標準の 4 スプリットを使用。VisualInstance3D の `ShadowCastingMode` は全ノードデフォルトの `ShadowCastingSetting.Shadows` に依存。

### 2. OmniLight3D / SpotLight3D（LIGH 光源）にも影を有効化

`LightingLoader.CreateLightNode()` (`LightingLoader.cs:274-276`):

| プロパティ | 値 | 説明 |
|---|---|---|
| `ShadowEnabled` | `true` | スポット・ポイントライトにも影 |
| `ShadowBias` | `0.005` | 方向性ライトよりやや大きめ |
| `ShadowNormalBias` | `0.02` | 法線バイアス |

### 3. アンビエントライトは影をオフ

`CellAmbientLight` (DirectionalLight3D, Energy=0.3) は Fill Light としてのみ機能するため、影はオフのまま。

### 4. バイアス値の考え方

FO3 → Godot の WorldScale = 0.015 により、1 FO3 単位 ≒ 1.5 cm。Godot 標準の ShadowBias (≈ 0.1) をそのまま使うと約 6.7 FO3 単位相当となり、影が大きく浮く（Peter Panning）。

そこでバイアスを WorldScale に比例して縮小:
```
ShadowBias ≈ 0.1 × 0.015 = 0.0015 → 0.001
ShadowNormalBias ≈ 1.0 × 0.015 = 0.015 → 0.01
```
Omni/Spot ライトはより鋭角な入射角のため、やや大きめの値に設定。

## 残課題

- **シャドウマップ解像度**: デフォルト 2048。必要に応じて project.godot の `rendering/lights_and_shadows/shadow_atlas_size` で 4096 に設定可能
- **シャドウアクネ / Peter Panning**: バイアス値は暫定。実際のレンダリング結果を見て調整が必要
- **パフォーマンス**: 多数の OmniLight3D の影は GPU 負荷が高い。屋内（メガトン内）では問題になりにくいが、屋外ワールドではフィルタリングが必要かも
- **Contact Shadow**: Godot 4 の Contact Shadow（近接影補完）を有効にすると近距離の画質が向上する
- **SDFGI**: 大規模屋外ワールドでは SDFGI の導入も検討
