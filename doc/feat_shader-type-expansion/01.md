# シェーダータイプ拡張: Wing (29), SnowShader (14), ZBufferWrite (5)

## 問題

`NIFMaterialBuilder.cs` のシェーダータイプディスパッチで、以下の 3 タイプが未実装だった:

| 定数 | 値 | 説明 | 元の動作 |
|------|-----|------|---------|
| ShaderTypeWing | 29 | 半透明の羽根 | default → BuildDefault |
| ShaderTypeSnowShader | 14 | 積雪エフェクト | default → BuildDefault |
| ShaderTypeZBufferWrite | 5 | 特殊深度処理 | default → BuildDefault |

## 修正内容

### 1. BuildWing (type 29)

**半透明羽根シェーダー。** 以下の設定を適用:

- `Transparency = Alpha` — アルファブレンド有効
- `CullMode = Disabled` — 両面レンダリング（薄い羽根を表裏なく表示）
- `SpecularMode = Disabled` — スペキュラーオフ
- Diffuse (Slot 0) + Normal (Slot 1) を通常通りロード

### 2. BuildSnowShader (type 14)

**積雪シェーダー。** 以下の設定を適用:

- `Roughness = 0.8` — つや消しの雪面
- `Metallic = 0.0`, `SpecularMode = Disabled`
- `AlbedoColor = (1.3, 1.3, 1.35)` — 明るくわずかに青みがかった白色にティント
- Diffuse (Slot 0) + Normal (Slot 1) を通常通りロード

### 3. BuildZBufferWrite (type 5)

**ZBuffer 深度書き込み制御シェーダー。** 以下の設定を適用:

- 標準的な不透明マテリアルとして描画
- ShaderFlags ビット 0 (Specular) に応じて SpecularMode を制御
- Diffuse (Slot 0) + Normal (Slot 1) を通常通りロード
- カスタムシェーダーによる深度制御は今後の課題

## nif.xml との対応

プロジェクトのシェーダータイプ番号は nif.xml の `BSShaderType` 列挙体とは異なる。プロジェクトの番号付けは実際の FO3 NIF ファイルの実測値に基づく。
